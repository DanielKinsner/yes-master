## Why This Repo Is Closing

The app has become real enough that the old handoff/spec pile is now a source
of drift. It contains useful history, but too many documents describe slices,
plans, and handoffs that are no longer the clean product truth.

The new repo should preserve git history for recovery/debugging, but its working
docs should be rebuilt around the current program state and the new product
canon instead of requiring agents to spelunk old handoffs.

## Product Direction Captured Before Migration

- YES Master is not a low-bar private toy. Treat it as a private-solid product
  before any public release conversation.
- The canonical product is a serious local mastering assistant for finished
  tracks and albums.
- Suno/already-mastered material is not the canonical product purpose, but it
  is an important stress-test class because YES Master must not quietly make
  already-processed inputs worse.
- The app may let a user overcook a track. That is acceptable when the metering,
  warning, and export-review UX make the consequence visible.
- Export should remain user-owned. Technical failures can block export; taste
  and quality warnings should ask for review and still allow Export Anyway.
- Main UI owns creative shaping: preset, compression, EQ, tone, intensity.
- Right rail owns "be smart with your track": quality checks, delivery profile,
  advanced controls, compressor detail, ceiling, bit depth, and export review.
- Current compressor "Auto" language is misleading. It is preset/density
  fallback, not track-aware analysis. Rename current behavior to Preset and add
  explicit compressor modes: Preset, Manual, Off.
- Compressor Off should bypass the creative/preset compressor stage only. It
  must not disable limiter, ceiling protection, LUFS landing, metering, or
  export warnings.

## Recon Against Current Program State

Verified current behavior before migration:

- `npm test` passes.
- `npm run build` passes.
- `cargo test --lib` passes from `src-tauri`.
- Full `cargo test` passes from `src-tauri`.
- `npm run build:windows` passes and produces MSI/NSIS bundles.
- `cargo fmt --check` still has pre-existing Rust formatting drift.
- `cargo clippy --all-targets -- -D warnings` could not run because the Clippy
  component is not installed in this local Rust toolchain.

Targeted real-fixture snapshot:

- The slow fixture snapshot imports, analyzes, renders Universal, re-analyzes
  the output, and prints objective deltas.
- It passed mechanically, but the local already-processed fixture became much
  louder and flatter after default Universal render.
- Source measured about -14.6 LUFS integrated, -4.0 dBTP true peak, and 5.2 LU
  dynamic range.
- Rendered output measured about -8.4 LUFS integrated, -0.7 dBTP true peak, and
  3.5 LU dynamic range.
- Export checks warned for streaming headroom and low dynamic range.
- Interpretation: the app does warn, but current UX still makes it too easy to
  treat an overcooked export as a normal success path.

## Jump Fixes For The New Repo

1. Export review flow.
   - When checks warn, the primary export affordance should become
     `Export With Review`.
   - Clicking it should open a review/confirm panel with the warning rows and
     actions: `Adjust Settings` and `Export Anyway`.
   - Do not silently hand the user a questionable master with a proud success
     face.

2. Compressor mode.
   - Rename current Auto labels/readouts to Preset.
   - Add mode: Preset / Manual / Off.
   - Keep limiter, ceiling, LUFS landing, and warnings active in all modes.

3. Already-mastered input regression matrix.
   - Add a private-fixture protocol that records source metrics, export metrics,
     deltas, and warning codes across Universal, Loud, Clarity, and compressor
     Off.
   - This is the objective net for "exports sound worse than imports."

4. Realtime sweep gate.
   - Run the aggressive playback sweep from `docs/DANDOFF_2026-05-20.md`.
   - Verify Intensity, EQ, output gain, compressor threshold/density with
     Preview LUFS off and on.
   - If clean, remove the temporary diagnostic counters.

5. New repo identity cleanup.
   - Rename package/Cargo identity away from `album-mastering-studio-*`.
   - Product/app title remains `YES Master`.
